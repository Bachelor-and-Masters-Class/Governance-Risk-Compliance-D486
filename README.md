Overview

This repository contains my completed submission for DFN1 Task 1: Security System Evaluation and Remediation.

In this project, I assumed the role of a Chief Information Security Officer (CISO) responsible for reviewing a third-party Security Assessment Report (SAR) for Fielder Medical Center (FMC). The objective was to analyze security gaps, evaluate risks associated with specific security controls, and develop a remediation strategy aligned with federal cybersecurity standards.

The analysis focuses on improving the organization’s cybersecurity governance and ensuring compliance with Federal Information Security Management Act (FISMA) requirements using guidance from NIST Special Publication 800-53 Rev. 5.

Scenario

Fielder Medical Center implemented a new digital system to manage doctor licensing documentation and professional credentials. Physicians upload licensing artifacts containing sensitive personally identifiable information (PII) such as:

Legal names
Home addresses
Social Security numbers
Professional certification documentation

This system must also be accessible by government auditors to verify compliance for continued federal funding.

Due to security concerns, FMC hired an external consulting firm to perform a security assessment. The report identified several weaknesses in the organization's security controls, risk management processes, and compliance posture.

Project Objectives

The purpose of this assignment was to evaluate the findings of the external security assessment and develop a cybersecurity strategy to strengthen FMC’s security posture.

Key objectives included:

Identifying gaps in the organization’s security framework
Evaluating the risks associated with identified security controls
Justifying remediation decisions based on federal cybersecurity standards
Developing remediation strategies aligned with NIST guidance
Creating a PCI DSS–compliant security policy for new payment systems

Security Controls Evaluated

The assignment required analysis of five security controls identified in the Security Assessment Report:

AC-6 – Least Privilege
CA-5 – Plan of Action and Milestones (POA&M)
CA-7 – Continuous Monitoring
RA-3 – Risk Assessment
RA-7 – Risk Response

Each control was evaluated to determine risk level, potential impact, and whether remediation was required to maintain compliance with federal cybersecurity regulations.

Remediation Strategy

A remediation plan was developed for each identified control. The plan included:

Required security tools and technologies
Policy updates and governance improvements
Organizational changes
Implementation actions aligned with federal security frameworks

Examples of remediation measures include:

Implementing Role-Based Access Control (RBAC) and multifactor authentication (MFA)
Establishing a formal Plan of Action and Milestones (POA&M)
Deploying continuous monitoring tools such as SIEM and vulnerability scanners
Conducting updated risk assessments following major system changes
Creating a formal risk response framework

PCI DSS Security Policy

The project also required development of a PCI DSS–compliant policy addressing security concerns related to the implementation of a new point-of-sale system for medical equipment sales.

The policy includes controls for:
Firewall configuration and network segmentation
Removal of vendor-supplied default credentials
Deployment and monitoring of antivirus protection

Roles and responsibilities were defined for IT administrators, security analysts, and executive leadership to ensure compliance with PCI DSS requirements.
